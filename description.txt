4. Core Features & Deterministic Rules

4.1 Daily Habits

Matrix view (habits Г— days) with вњ“ (completed) and вњ— (skipped).
Automatic skip for any habit not logged by the end of the day.
Streak tracking: longest streak, current streak, average skip days.
4.2 ShortвЂ‘Term Tasks

Each task has: text, assigned date, status, rollover count, priority (1вЂ‘3).
Daily rollover automatically moves pending tasks to the next day and increments rolloverCount.
If rolloverCount >= 3 в†’ suggestion to split the task.
4.3 Projects

A project is a structured goal of any length with an optional due date (recommended but not required).
Projects are divided into Phases (e.g. Research, Planning, Backend, Frontend).
Each Phase contains its own ordered list of Milestones.
Milestones are sequential вЂ” next one cannot start until previous is completed.
Milestones can be added manually by the user or suggested by the system (user must approve system suggestions).
If a milestone is not completed on time, the system notices and suggests recalibrating the plan.
A progress bar shows how far along the project is (completed milestones / total milestones).
Later: estimated completion date based on current pace.
UI vision: Duolingo-style vertical ladder вЂ” Project at the top, Phases as sections, Milestones as steps within each phase. Design to be figured out during frontend week.

4.4 Long-Term Goals [DEFERRED]

Concept is on hold until core app is built. Similar structure to Projects but with 90-day planning cycles.
Key question to answer later: is this different enough from Projects to warrant a separate feature, or just a project with a planning mode?
Do not discard вЂ” revisit after frontend is complete.
4.4 Deterministic Suggestion Engine (No AI)

Split task suggestion вЂ“ triggered by tasks with rolloverCount в‰Ґ 3.
Productivity tip вЂ“ if productivity rate < 0.3, suggest limiting daily tasks.
Completion likelihood prediction вЂ“ formula based on user productivity, task difficulty, and todayвЂ™s task count.
Productivity rate formula:

text
taskScore   = completedTasks / totalTasks (last 30 days)
habitScore  = completedHabits / totalHabitLogs
productivity = 0.6 * taskScore + 0.4 * habitScore
Completion likelihood:

text
likelihood = productivity * (1 - difficulty) * (1 / (todayTaskCount + 1))
difficulty = 0.7 for priority 3, 0.5 for priority 2, 0.3 for priority 1
5. Database Schema (Prisma)

prisma
model User {
  id        String     @id @default(uuid())
  email     String     @unique
  name      String
  createdAt DateTime   @default(now())
  habits    Habit[]
  tasks     ShortTask[]
  projects  Project[]
}

model Habit {
  id        String     @id @default(uuid())
  userId    String
  name      String
  isActive  Boolean    @default(true)
  createdAt DateTime   @default(now())
  logs      HabitLog[]
  user      User       @relation(fields: [userId], references: [id], onDelete: Cascade)
}

model HabitLog {
  id        String   @id @default(uuid())
  habitId   String
  date      String   // YYYY-MM-DD
  status    String   // "completed" | "skipped" | "excused"
  notes     String?
  habit     Habit    @relation(fields: [habitId], references: [id], onDelete: Cascade)
}

model ShortTask {
  id            String   @id @default(uuid())
  userId        String
  text          String
  dateAssigned  String
  status        String   // "pending" | "completed" | "do_it_today" | "not_actual"
  rolloverCount Int      @default(0)
  priority      Int      @default(1)
  createdAt     DateTime @default(now())
  user          User     @relation(fields: [userId], references: [id], onDelete: Cascade)
}

model Project {
  id            String      @id @default(uuid())
  userId        String
  title         String
  description   String?
  targetEndDate String
  createdAt     DateTime    @default(now())
  milestones    Milestone[]
  user          User        @relation(fields: [userId], references: [id], onDelete: Cascade)
}

model Milestone {
  id          String   @id @default(uuid())
  projectId   String
  title       String
  order       Int
  completed   Boolean  @default(false)
  dueDate     String?
  project     Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)
}

9.1 AI Planning Assistant [DEFERRED вЂ” Post MVP]

Vision: User types a goal in plain text (e.g. "I want to learn React in 2 months").
AI generates a full project plan вЂ” phases and milestones вЂ” as a preview.
User approves or rejects. If approved, the plan is inserted into the database using existing API endpoints.

How it works:
- New endpoint: POST /api/ai/suggest вЂ” accepts a text prompt
- Sends prompt to Claude API (or similar LLM) with instructions to respond in JSON matching the project schema
- Frontend shows the generated plan as a preview card
- User clicks Approve в†’ frontend calls existing createProject, createPhase, createMilestone endpoints
- User clicks Reject в†’ nothing is saved

Key point: the backend API is already built. AI just generates the data that fills it.
When to build: after frontend is complete and deployed. Requires Anthropic API key.
