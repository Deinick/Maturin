Autonomous Productivity Planner – Master Architecture Document

1. Project Overview

A full‑stack productivity web application that helps users track daily habits, manage short‑term tasks, and break down long‑term projects into visual “ladders” of milestones. The system provides deterministic, rule‑based suggestions (e.g., splitting stalled tasks, productivity tips) without relying on external AI APIs. All logic is custom‑coded.

Target platforms:

Web app (primary, deployed)
Desktop app (later – Electron)
Mobile app (later – Expo / React Native)

4. Core Features & Deterministic Rules

4.1 Daily Habits

Matrix view (habits × days) with ✓ (completed) and ✗ (skipped).
Automatic skip for any habit not logged by the end of the day.
Streak tracking: longest streak, current streak, average skip days.
4.2 Short‑Term Tasks

Each task has: text, assigned date, status, rollover count, priority (1‑3).
Daily rollover automatically moves pending tasks to the next day and increments rolloverCount.
If rolloverCount >= 3 → suggestion to split the task.
4.3 Project Ladder

A project contains a sequential list of milestones (ordered).
Milestones are shown as a vertical “ladder” with progress bar.
Next milestone cannot be completed until the previous one is done.
4.4 Deterministic Suggestion Engine (No AI)

Split task suggestion – triggered by tasks with rolloverCount ≥ 3.
Productivity tip – if productivity rate < 0.3, suggest limiting daily tasks.
Completion likelihood prediction – formula based on user productivity, task difficulty, and today’s task count.
Productivity rate formula:

text
taskScore   = completedTasks / totalTasks (last 30 days)
habitScore  = completedHabits / totalHabitLogs
productivity = 0.6 * taskScore + 0.4 * habitScore
Completion likelihood:

text
likelihood = productivity * (1 - difficulty) * (1 / (todayTaskCount + 1))
difficulty = 0.7 for priority 3, 0.5 for priority 2, 0.3 for priority 1
5. Database Schema (Prisma)

prisma
model User {
  id        String     @id @default(uuid())
  email     String     @unique
  name      String
  createdAt DateTime   @default(now())
  habits    Habit[]
  tasks     ShortTask[]
  projects  Project[]
}

model Habit {
  id        String     @id @default(uuid())
  userId    String
  name      String
  isActive  Boolean    @default(true)
  createdAt DateTime   @default(now())
  logs      HabitLog[]
  user      User       @relation(fields: [userId], references: [id], onDelete: Cascade)
}

model HabitLog {
  id        String   @id @default(uuid())
  habitId   String
  date      String   // YYYY-MM-DD
  status    String   // "completed" | "skipped" | "excused"
  notes     String?
  habit     Habit    @relation(fields: [habitId], references: [id], onDelete: Cascade)
}

model ShortTask {
  id            String   @id @default(uuid())
  userId        String
  text          String
  dateAssigned  String
  status        String   // "pending" | "completed" | "do_it_today" | "not_actual"
  rolloverCount Int      @default(0)
  priority      Int      @default(1)
  createdAt     DateTime @default(now())
  user          User     @relation(fields: [userId], references: [id], onDelete: Cascade)
}

model Project {
  id            String      @id @default(uuid())
  userId        String
  title         String
  description   String?
  targetEndDate String
  createdAt     DateTime    @default(now())
  milestones    Milestone[]
  user          User        @relation(fields: [userId], references: [id], onDelete: Cascade)
}

model Milestone {
  id          String   @id @default(uuid())
  projectId   String
  title       String
  order       Int
  completed   Boolean  @default(false)
  dueDate     String?
  project     Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)
}

10. Key Design Decisions & Why

Decision	Why	Alternative (rejected)
TypeScript	Type safety, early error detection	Plain JS – runtime crashes
PostgreSQL	Production ready, concurrent access	SQLite – not suitable for cloud
Prisma	Type‑safe ORM, auto migrations	Raw SQL – error‑prone
No AI APIs	Full control, no costs, deterministic	LLM wrapper – black box, costs
Tailwind CSS	Fast UI development	Plain CSS – slower
Express	Simple, flexible	NestJS – overkill
Monorepo (separate folders)	Clear separation, independent deployments	Combined – harder to manage

15. Glossary

Term	Meaning
ORM	Object‑Relational Mapper – maps database tables to code objects
Migration	Script that changes database schema (add table, column, etc.)
Rollover	Moving uncompleted tasks to the next day
Deterministic	Given same input, always produces same output (no randomness)
Ladder	Visual representation of sequential milestones
Multi‑tenancy	One database serving multiple users, each seeing only their data
